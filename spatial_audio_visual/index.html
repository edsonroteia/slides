Spatial Answer
Where is each object?

Track from the camera's viewpoint using three complementary signals:

a
Ask Gemini

One structured prompt identifies object roles (target, reference, facing), grounds the time window, and outputs a sparse trajectory. Good starting point, but misses frames.

b
Segment + Depth

Segment objects across sampled frames (CLIPSeg + SAM2). Angle from mask centroid offset to image center; distance from monocular depth (ZoeDepth). Fills the trajectory gaps.

c
Spatial audio

7-mic array on the Aria glasses. Angle from beam-steering across mic pairs; distance from coherent-to-diffuse ratio, calibrated with the visual estimates. Only signal that works behind the camera.

Each signal outputs (time, angle, distance) per object per frame
Build the overhead map

Merge all camera views into one bird's-eye floor plan:

1
Project to global coordinates

Use the camera trajectory to place each observation on a shared 2D map

2
Clean up the tracks

Pin down static objects by clustering, smooth moving trajectories

3
Answer the question

Read direction & distance from the map, rotate the frame if allocentric

Result: a unified map with all object positions over time
Key result: Global mapping gives +11.9% localization, +14.3% distance, +4.1% direction over raw egocentric tracks.

Failure Mode 3
Distance Estimation: Regression Collapse
QWEN3-OMNI
MINICPM-o-4.5
Regression Collapse (Both Models)

Both models show constant predictions for ego-distance (clustered around 0.8-1.5m). MRA scores are misleading: for indoor scenes, even a constant can score well.

Shared Failure Patterns

Predictions cluster at integer values. Faint positive correlation for GT 2-5m, but hard ceiling at ~8m where GT extends to 12m. Range compression is universal across both models.

SAVVY fix: ZoeDepth provides calibrated per-frame metric depth. CDR provides audio-based distance. Both give actual metric signals.
MRA = 1/10 ∑t=0.11.0 𝟙[ |ŷ − y| ≤ t ]  thresholds: 0.1m … 1.0m

Next Steps
Other Research Directions

Two complementary directions targeting the core failure modes, not exploring generation at this stage.

Direction 1 Fixes: Front-Bias · Distance Collapse
Spatial Audio Encoder Integration

Incorporate a spatial audio encoder (e.g., multi-channel / binaural) into the model architecture, followed by lightweight adapter training to bridge the audio encoder to the language model.

Approach: Small-scale training: freeze the LLM, train only the adapter layers to project spatial audio features into the model's embedding space.

Expected impact: Behind-camera localization (front-bias) and metric distance estimation.

Direction 2 Fixes: Allocentric Collapse
Allocentric Perspective Transformation

Explore how models can perform ego-to-allocentric coordinate changes, the ability to reason about spatial relationships from a third-person reference frame.
